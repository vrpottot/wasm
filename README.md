# Pixi.js to Skia CanvasKit Vector PDF Renderer

Веб-приложение на **TypeScript**, демонстрирующее интеграцию графического движка **Pixi.js** и **Google Skia (CanvasKit/WASM)** с поддержкой двустороннего рендеринга, трансляции событий ввода (pointer events), рекурсивного hit-testing на матричных трансформациях и экспорта сцены в векторный **PDF** через Skia PDF backend.

## 🚀 Основные возможности

1. **Dual Renderers Mirroring**: Отрисовка сцены PixiJS (WebGL/Canvas) и Skia (CanvasKit) бок о бок с полной синхронизацией матричных трансформаций (сдвиг, поворот, масштаб, точки привязки/anchor) и прозрачности.
2. **Interactive Event Bridge**: Проброс событий мыши/касания (`pointerdown` / `pointerup`) с Skia-канваса обратно на объекты PixiJS с помощью рекурсивного hit-testing'а в локальном пространстве координат каждого объекта (`worldTransform.applyInverse`).
3. **Vector PDF Export**: Прямой экспорт сцены в формате PDF. Фигуры и контуры экспортируются как векторные пути (а не растровые картинки), обеспечивая идеальную четкость при любом масштабировании в Adobe Acrobat, Chrome, macOS Preview и др. Растровые спрайты (`PIXI.Sprite`) вшиваются как bitmap.
4. **Интерактивные инструменты**: Добавление случайных фигур с разнообразными цветами, альфой и случайными трансформациями в иерархическое дерево сцены в реальном времени.

## 🛠 Технологический стек

* **Язык**: TypeScript (строгий режим `strict: true`)
* **Сборщик**: Vite
* **Рендеринг**:
  * `pixi.js-legacy` версии `^7.2.4` (для поддержки canvas-рендеринга и управления событиями сцены)
  * `@rollerbird/canvaskit-wasm-pdf` (кастомная WASM-сборка Google Skia с включенной поддержкой PDF backend)
* **Стилизация**: Чистый CSS с использованием кастомных HSL-палитр, размытия стекла (glassmorphism) и анимаций.

## 💻 Установка и запуск

### Требования
* **Node.js** версии `>= 18.0.0`
* **NPM** версии `>= 9.0.0`

### Шаги для локального запуска

1. **Клонирование репозитория**:
   ```bash
   git clone <ссылка_на_ваш_репозиторий>
   cd <папка_проекта>
   ```

2. **Установка зависимостей**:
   ```bash
   npm install
   ```

3. **Запуск в режиме разработчика**:
   ```bash
   npm run dev
   ```
   Приложение откроется по адресу [http://localhost:3000](http://localhost:3000).

4. **Сборка для продакшена**:
   ```bash
   npm run build
   ```
   Собранные оптимизированные файлы будут находиться в папке `dist/`.

## ⚙️ Сборка кастомного CanvasKit с PDF backend

По умолчанию официальная сборка `canvaskit-wasm` на NPM не содержит PDF-модуля для уменьшения веса WASM-файла. В данном проекте используется готовая сборка `@rollerbird/canvaskit-wasm-pdf`.

Если вы хотите собрать WASM самостоятельно из исходного кода Google Skia, выполните следующие шаги:

1. **Настройка окружения**:
   * Установите [Emscripten SDK](https://emscripten.org/docs/getting_started/downloads.html) и активируйте его в терминале.
   * Установите Python 3, Git, Ninja и GN.

2. **Клонирование репозитория Skia**:
   ```bash
   git clone https://skia.googlesource.com/skia.git
   cd skia
   python tools/git-sync-deps
   ```

3. **Сборка CanvasKit с включением PDF**:
   Сборка CanvasKit запускается с помощью сборочного скрипта `modules/canvaskit/compile.sh`. Передайте флаг для включения PDF:
   ```bash
   # В терминале с активированным Emscripten (emsdk):
   ./modules/canvaskit/compile.sh --extra-gn-args="skia_use_pdf=true"
   ```
   
4. **Копирование результатов**:
   После завершения компиляции скомпилированные файлы (`canvaskit.js` и `canvaskit.wasm`) будут находиться в `out/canvaskit`.
   Скопируйте их в папку `/public/canvaskit` вашего Vite-проекта.

## 📂 Архитектура проекта

```
src/
├── pixi/
│   └── scene.ts          # Инициализация приложения PixiJS, создание тестовой сцены, спрайтов и логирование
├── skia/
│   ├── renderer.ts       # Рекурсивный обход дерева Pixi-контейнеров, отрисовка графики и спрайтов через CanvasKit
│   └── pdf-exporter.ts   # Создание векторного PDF-документа, рендер сцены в PDF и скачивание файла
├── events/
│   └── event-bridge.ts   # Hit-testing в локальном пространстве координат и проброс событий pointerdown/pointerup
├── ui/
│   └── (управляется через main.ts и index.html)
├── index.css             # Стили и дизайн-система приложения
└── main.ts               # Точка входа, связывающая загрузку WASM, рендеринг, события и элементы интерфейса
```

## ⚠️ Известные ограничения

* **Спрайты в PDF**: Все `PIXI.Sprite` экспортируются в PDF как растровые изображения (PNG), что допускается спецификацией (поскольку сами исходники спрайтов являются растровыми). Фигуры `PIXI.Graphics` экспортируются строго в векторе.
* **Шрифты**: Текстовые надписи в текущей версии рендерятся средствами PixiJS. Для поддержки векторного текста в Skia/PDF требуется загрузка кастомного шрифта TTF/WOFF через `canvaskit.Typeface.MakeTypefaceFromData()`.
