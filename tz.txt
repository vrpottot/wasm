# Техническое задание: TypeScript + Pixi.js + Skia → PDF

---

## 1. Общее описание проекта

Необходимо разработать веб-приложение на **TypeScript**, которое:

1. Рендерит сцену Pixi.js через движок **Skia (CanvasKit/WASM)**
2. Поддерживает интерактивные события мыши/касания
3. Экспортирует сцену в **векторный PDF** через Skia PDF backend

Приложение должно запускаться локально через `npm run dev` (или аналог), быть задеплоено на бесплатный хостинг и опубликовано на GitHub.

## 4. Задача 1 — Обёртка Skia для Pixi.js

### 4.1 Интерфейс

Нужно реализовать функцию-конвертер, которая принимает `PIXI.Container` и отрисовывает его содержимое на Skia Canvas:

```typescript
const convertPixiContainerToSkia = (
  container: PIXI.Container,
  skiaCanvas: CanvasKit.Canvas
): void
```

### 4.2 Поддерживаемые объекты

**PIXI.Graphics** — необходимо поддержать следующие методы:
- `beginFill(color)` / `endFill()`
- `lineStyle(width, color, alpha)`
- `drawRect(x, y, width, height)`
- `drawEllipse(x, y, rx, ry)`
- `drawShape(...)`
- `moveTo(x, y)` / `lineTo(x, y)`

**PIXI.Sprite** — растровые изображения (PNG). Отрисовываются как bitmap (не вектор), что является допустимым исключением при PDF-экспорте.

**PIXI.Container** — рекурсивный обход дерева вложенных контейнеров.

### 4.3 Трансформации

Для каждого объекта сцены необходимо корректно применять трансформации. Матрица вычисляется из:

| Свойство Pixi | Описание |
|---|---|
| `position.x / position.y` | Смещение (translate) |
| `angle` | Поворот в градусах |
| `scale.x / scale.y` | Масштабирование |
| `pivot.x / pivot.y` | Точка привязки |

Трансформации должны применяться с учётом иерархии — то есть трансформация родительского контейнера влияет на дочерние объекты (мировая матрица = произведение всех матриц вверх по дереву). Для этого используется `skiaCanvas.save()` / `skiaCanvas.restore()` при рекурсивном обходе.

### 4.4 Пример сцены для тестирования

```typescript
const mainContainer = new PIXI.Container()
const subContainer = new PIXI.Container()

const g1 = new PIXI.Graphics()
g1.beginFill('#ff0000').drawEllipse(0, 0, 200, 100).endFill()
g1.position.set(200, 100)
g1.angle = 30

const g2 = new PIXI.Graphics()
g2.beginFill('#0000ff').drawRect(-50, -75, 100, 150).endFill()
g2.position.set(120, 60)
g2.angle = 15
g2.scale.set(1.5, 1.7)

const g3 = new PIXI.Graphics()
g3.lineStyle(10, '#ffffff', 1).moveTo(0, 0).lineTo(150, 100)
g3.angle = -20

const g4 = new PIXI.Graphics()
g4.lineStyle(10, '#ffff00', 1).moveTo(0, 70).lineTo(150, -30)
g4.angle = 20

subContainer.position.set(75, 50)
subContainer.addChild(g3, g4)
mainContainer.addChild(subContainer, g1, g2)
```

Результат рендера Skia должен визуально совпадать с Pixi.js-рендером.

## 5. Задача 2 — Экспорт в векторный PDF

### 5.1 Требования к PDF

- Результат должен быть **векторным** — фигуры, нарисованные через `PIXI.Graphics`, должны экспортироваться как векторные пути, а не как растровое изображение
- `PIXI.Sprite` допускается экспортировать как bitmap (исключение из векторного требования)
- Итоговый PDF должен корректно открываться в Adobe Acrobat, браузере, macOS Preview и других стандартных просмотрщиках

### 5.2 Реализация

Используется **Skia PDF backend** — `CanvasKit.MakePDFDocument()`. Процесс экспорта:

```
1. Создать PDF-документ: CanvasKit.MakePDFDocument()
2. Добавить страницу с размерами сцены: doc.beginPage(width, height)
3. Вызвать convertPixiContainerToSkia(container, pdfCanvas)
4. Завершить страницу: doc.endPage()
5. Сериализовать: doc.close() → Uint8Array
6. Сохранить файл через Blob + <a download>
```

### 5.3 Кастомная WASM-сборка

Стандартная сборка CanvasKit не включает PDF backend. Необходимо:

1. Клонировать репозиторий Skia
2. Собрать CanvasKit с флагом `skia_use_pdf=true` через `gn` + `ninja`
3. Подключить полученный `canvaskit.wasm` в проект

В README должна быть подробная инструкция по сборке WASM либо предоставлен готовый файл в репозитории.

## 6. Задача 3 — Интерактивность и события

### 6.1 Поддержка событий

Необходимо обеспечить работу событий `pointerdown` и `pointerup` для объектов `PIXI.DisplayObject` на **обоих канвасах** — Pixi и Skia.

Логика:
- Pixi обрабатывает события нативно
- Для Skia-канваса нужно реализовать собственный hit-testing: при клике проверяется, попадает ли координата курсора (с учётом трансформаций объекта) в его bounding box / контур

```typescript
skiaCanvas.addEventListener('pointerdown', (e) => {
  const worldPoint = { x: e.offsetX, y: e.offsetY }
  // Обход дерева объектов, проверка hit-test
  // Вызов соответствующего обработчика PIXI-объекта
})
```

### 6.2 Интерактивный функционал (на выбор один вариант)

**Вариант A — Генерация случайных фигур:**
Кнопка «Добавить фигуру» добавляет в текущий контейнер случайный `PIXI.Graphics` — прямоугольник, эллипс или линию — со случайными координатами, размерами, цветом и углом поворота. Оба канваса обновляются.

**Вариант B — Переключение контейнеров:**
Подготовить 3–5 различных `PIXI.Container` с разным содержимым. Переключение между ними — по кнопкам или по таймеру (`setTimeout`). При переключении оба канваса перерисовываются.

## 7. UI приложения

Интерфейс должен включать:

- **Два канваса** рядом или один под другим: слева/сверху — Pixi.js, справа/снизу — Skia
- **Подписи** к канвасам («Pixi.js Renderer» / «Skia Renderer»)
- **Кнопка «Экспорт в PDF»** — инициирует скачивание PDF-файла
- **Кнопка интерактивности** — согласно выбранному варианту из п. 6.2
- **Индикатор событий** — текстовая строка, показывающая последнее сработавшее событие (например: `g1: pointerdown`, `g2: pointerup`)

Пример минимальной разметки:


## 8. Технические требования к коду

- Весь код на **TypeScript** со строгой типизацией (`strict: true` в `tsconfig.json`)
- Модульная архитектура — каждый модуль отвечает за одну задачу
- Комментарии на русском или английском языке для нетривиальных частей кода (особенно матричные вычисления и hit-testing)
- Нет магических чисел — все константы вынесены в отдельный файл или именованные переменные
- Обработка ошибок при загрузке WASM и при экспорте PDF
